# Friend Recommend System 项目结构与代码功能说明

本文档按“目录 -> 文件 -> 主要职责/关键逻辑”说明当前项目代码结构，便于后续维护、交接和二次开发。面向新环境的**安装、数据库初始化与启动步骤**以仓库根目录的 [README.md](./README.md) 为准；本文侧重结构与实现细节。

## 1. 项目总览

- `frontend/`: Vue 3 + Vite 前端，负责页面交互、路由守卫、状态管理、推荐/好友/聊天等业务界面。
- `backend/`: Flask + SQLite 后端，负责认证、资料、推荐计算（PSI + Jaccard）、好友关系、聊天接口。
- 数据库文件默认位于 `backend/database/database.db`（由 `init_db.py` 创建，**勿将含真实数据的 `.db` 提交到公开仓库**；仓库通过 `.gitignore` 忽略该文件）。

### 1.1 仓库根目录文件（与 GitHub 发布相关）

| 文件 | 作用 |
|------|------|
| `requirements.txt` | 后端 Python 依赖：`Flask`、`numpy`；`Faker` 供 `seed_data.py` 造数使用。安装：`pip install -r requirements.txt`（建议在项目根目录执行）。 |
| `package.json` | 根级脚本封装，通过 `npm --prefix frontend` 调用子项目，便于在根目录执行 `npm run frontend:dev` 等（详见 README）。 |
| `README.md` | 对外说明：环境要求、SQLite 初始化、前后端启动命令、环境变量、部署注意。 |
| `.gitignore` | 忽略 `node_modules/`、虚拟环境、`backend/database/*.db`、`frontend/dist/` 等，避免误提交依赖与本地数据库。 |

### 1.2 依赖版本关系（摘要）

- **后端**：`Flask` 提供 HTTP 与会话；`Werkzeug`（随 Flask）负责密码哈希；`numpy` 在 `modules/recommend.py` 中参与推荐相关数值运算；`sqlite3` 为标准库。
- **前端**：依赖声明在 `frontend/package.json`（`vue`、`vue-router`、`pinia`、`vite`、`@vitejs/plugin-vue`、`@heroicons/vue`、`@tanstack/vue-virtual` 等），与根目录 `package.json` 互补——**实际安装仍在 `frontend/` 下执行** `npm install`。

## 2. 前端结构（`frontend/`）

### 2.0 构建与代理

- `frontend/vite.config.js`
  - 开发服务器端口 `5173`；`server.proxy` 将 `/api` 转发到 `http://127.0.0.1:5000`（Flask）。
  - `preview` 同样配置 `/api` 代理，便于在 `npm run build` 后本地预览时仍能访问后端。
- `frontend/index.html`
  - HTML 壳：挂载 `#app`、引入 `src/main.js`、页面标题与 Google Fonts（Noto Sans SC / Nunito）。

### 2.1 入口与基础层

- `frontend/src/main.js`
  - 前端应用入口。
  - 创建 Vue App，挂载 Pinia 与 Vue Router。
  - 引入全局样式 `global.css`。

- `frontend/src/App.vue`
  - 根组件，负责：
    - 渲染页面路由容器（`RouterView`）；
    - 页面切换淡入淡出过渡；
    - 全局浮动 toast 的 Teleport 容器。

- `frontend/src/styles/global.css`
  - 全局设计令牌与通用样式：
    - 主题色、模式色（好友/组队/恋爱）、边框、字号、阴影等 CSS 变量；
    - 按钮、输入框、toast、基础排版。

### 2.2 API 与状态管理

- `frontend/src/api/client.js`
  - 统一 `fetch` 封装：
    - 请求路径使用以 `/api` 开头的相对路径，开发/预览下由 Vite 代理到 Flask；
    - 自动 JSON 序列化/反序列化；
    - 自动携带 cookie（`credentials: include`）；
    - 统一错误解析与抛出。

- `frontend/src/stores/auth.js`
  - 认证状态仓库（Pinia）：
    - 当前用户态 `user`、初始化状态 `ready`；
    - 账号登录、手机号登录、短信发送、找回密码、注册、退出登录；
    - 启动时 `fetchMe()` 恢复会话。

- `frontend/src/stores/ui.js`
  - UI 状态仓库：
    - 全局 toast 队列；
    - 去重策略（相同文案+类型不重复堆叠）；
    - 定时自动移除。

### 2.3 路由层

- `frontend/src/router/index.js`
  - 全量页面路由定义。
  - `/app/*` 路由配置 `requiresAuth`。
  - 前置守卫：
    - 首次导航自动拉取当前登录态；

## 3. 后端结构（`backend/`）

### 3.1 入口与配置

- `backend/app.py`
  - Flask 入口：
    - 注册各业务蓝图；
    - 处理 CORS（含预检 OPTIONS）；
    - 提供 `/` 与 `/health` 诊断接口。

- `backend/config.py`
  - 统一配置：
    - 后端根目录；
    - SQLite 数据库路径；
    - 前端地址与允许跨域来源列表。

### 3.2 认证与用户摘要

- `backend/modules/user.py`
  - 认证相关 API：
    - 账号密码登录；
    - 手机验证码登录；
    - 演示短信验证码发送（固定 `123456`）；
    - 找回密码重置；
    - 注册（含可选手机号验证、兴趣初始化）；
    - 获取当前登录用户；
    - 登出清 session。
  - 统计摘要 API（`/api/me/summary`）：
    - 好友数、推荐次数、待处理请求、活跃度分。

### 3.3 资料与隐私偏好

- `backend/modules/profile.py`
  - 资料读写：
    - 读取完整 profile；
    - 更新昵称、基础字段、可选密码；
    - 批量保存兴趣/技能/项目/性格列表。
  - 隐私偏好：
    - `share_interests/share_skills/share_friend_graph` 开关存取。
  - 公开资料接口：
    - 根据关系（self/friend/stranger）返回分级信息。

### 3.4 推荐计算

- `backend/modules/recommend.py`
  - 推荐核心逻辑：
    - 三模式权重（好友/组队/恋爱）；
    - PSI 交集 + Jaccard 评分；
    - 按分排序、分页返回。
  - 详细推荐接口：
    - 返回综合分、维度分、推荐理由、兴趣/技能预览。
  - 推荐历史快照：
    - 首批请求自动落库；
    - 历史查询与单条删除。

- `backend/modules/psi.py`
  - 轻量级 PSI 原型实现（OKVS 思路）：
    - 哈希映射 + 掩码编码；
    - 交集查询接口 `psi_intersection`。

### 3.5 社交关系与聊天

- `backend/modules/friend.py`
  - 好友体系 API：
    - 好友列表读取；
    - ID 互加好友；
    - 好友请求创建、收件箱、发件箱、同意、拒绝。
  - 展示增强：
    - 演示在线态；
    - 最近互动与圈子活跃徽章。

- `backend/modules/chat.py`
  - 私信 API：
    - 与好友聊天权限校验；
    - 会话全量/增量消息拉取；
    - 发送消息；
    - 会话信息（如对方最近活跃）。

- `backend/modules/__init__.py`
  - 模块包标记文件（当前仅注释说明）。

### 3.6 数据库与数据脚本

- `backend/init_db.py`
  - 数据库初始化与迁移脚本：
    - 创建全部核心表；
    - 对旧 `users` 表进行 schema 迁移；
    - 明文密码升级为哈希存储。

- `backend/insert_test_data.py`
  - 写入小规模示例数据（便于本地功能联调）。

- `backend/seed_data.py`
  - 批量生成大规模模拟数据（1000 用户）用于推荐实验。

- `backend/experiment_recommend.py`
  - 离线推荐实验脚本：
    - 逐用户、逐模式跑推荐；
    - 输出交集维度与综合分日志。

- `backend/test_db.py`
  - 快速数据库连通与数据读取脚本（调试用途）。

**路径约定**：`backend/config.py` 与 `init_db.py` 均将库文件放在 `backend/database/database.db`；首次运行前在 `backend` 目录执行 `python init_db.py` 即可自动创建目录与文件。

## 4. SQLite 数据库说明

### 4.1 如何初始化（首次或重置）

1. 进入 `backend` 目录：`cd backend`。
2. 执行：`python init_db.py`。
3. 成功后终端会打印 `数据库已就绪:` 及绝对路径；若存在旧版 `users` 表结构，脚本会迁移并打印相应提示。

**重置数据库（慎用）**：停止后端进程后，删除 `backend/database/database.db`（及同目录下的 `-journal` 临时文件），再重新执行 `python init_db.py`，即可得到空表结构。之后可再运行 `insert_test_data.py` 或 `seed_data.py` 填充数据。

### 4.2 表结构一览（业务含义）

| 表名 | 用途 |
|------|------|
| `users` | 用户账号：`account` 登录名、`username` 显示名、`password` 为 Werkzeug 哈希；可选 `gender`/`grade`/`major`/`phone`。 |
| `friendships` | 已建立好友关系：`user_id`、`friend_id`、`created_at`。 |
| `user_interests` | 用户兴趣标签（推荐维度之一）。 |
| `user_skills` | 用户技能（组队类推荐）。 |
| `user_projects` | 项目/比赛经历。 |
| `user_traits` | 性格标签（恋爱类推荐）。 |
| `user_matching_prefs` | 隐私与匹配开关：`share_interests`、`share_skills`、`share_friend_graph`（0/1）。 |
| `friend_requests` | 好友请求：`from_id`、`to_id`、`status`（如 pending）、`created_at`。 |
| `recommendation_history` | 推荐结果快照：`user_id`、`mode`、`snapshot_json`、`created_at`。 |
| `messages` | 私信：`sender_id`、`receiver_id`、`body`、`created_at`。 |

### 4.3 迁移逻辑（init_db 摘要）

- 若检测到旧 `users` 表缺少 `account` 列：新建 `users_new`，把旧数据迁入（`account` 默认沿用原 `username`），密码在非哈希形式下会 `generate_password_hash` 后再写入，最后替换表名。
- `friendships` 若缺 `created_at` 列则 `ALTER TABLE` 补齐并回填当前时间。
- `users` 若缺 `phone` 列则 `ALTER TABLE` 添加，以支持手机验证码登录与找回密码演示流程。

## 5. 环境与启动（开发）

以下与 [README.md](./README.md) 一致，便于在本文内快速查阅。

1. **Python 虚拟环境**（根目录）：`python -m venv .venv` 并激活后，`pip install -r requirements.txt`。
2. **初始化库**（`backend`）：`python init_db.py`。
3. **前端依赖**（`frontend`）：`npm install`，或根目录 `npm run frontend:install`。
4. **终端 A — 后端**（`backend`）：`python app.py`（默认端口 5000）。
5. **终端 B — 前端**（根目录）：`npm run frontend:dev`（默认 5173，API 走代理）。

可选环境变量见 `backend/config.py` 与 `app.py`：`FRONTEND_URL`、`PORT`、`FLASK_SECRET_KEY`。

## 6. 运行关系与数据流

### 6.1 登录态链路

1. 前端 `auth.fetchMe()` 调 `/api/auth/me`。
2. 后端基于 session 返回当前用户信息。
3. 路由守卫据此判断是否放行受保护页面。

### 6.2 推荐链路

1. 前端推荐页调用 `/api/recommend/<mode>/detailed`。
2. 后端读取用户好友/兴趣/技能集合与隐私偏好。
3. 对候选用户执行 PSI 交集与 Jaccard 评分。
4. 返回排序结果；首批请求写入推荐历史快照。

### 6.3 聊天链路

1. 进入聊天页先拉全量消息。
2. 定时轮询 `after_id` 拉增量。
3. 前端更新本地已读游标与未读数展示。

## 7. 维护建议

- 生产环境建议将演示短信逻辑（固定验证码）替换为真实短信服务并加限流。
- 推荐计算当前以 SQLite + Python 循环实现，用户规模继续上升时建议：
  - 增加缓存与预计算；
  - 拆分在线计算与离线召回；
  - 增加索引与批处理策略。
- 聊天目前为轮询，若需更实时可升级为 WebSocket。
