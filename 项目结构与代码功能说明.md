### 2.2 PSI 隐私集合交集

#### `backend/modules/psi.py` — OKVS-PSI 协议实现

这是项目的**核心隐私保护模块**，基于 OKVS（Oblivious Key-Value Store）论文风格实现。

**核心概念：**

1. **PRF（伪随机函数）**
   ```python
   def _prf(secret, item):
       # 基于 SHA256 的确定性哈希
       h = hashlib.sha256((str(secret) + str(item)).encode()).hexdigest()
       return int(h[:16], 16)  # 取前16位十六进制字符转为整数
   ```
   - 输入：密钥 `secret` + 元素 `item`
   - 输出：伪随机的整数（确定性：相同输入产生相同输出）

2. **LightweightOKVS 类 — 多哈希 XOR 编码哈希表**
   ```python
   class LightweightOKVS:
       def __init__(self, m: int = 10007, k: int = 3):
           # m: 哈希表大小（建议选大素数，影响碰撞率）
           # k: 每个元素的哈希位置数（k 越大碰撞越少）
           self.table = [0] * m  # 初始化全0哈希表
           self.salts = [str(i) for i in range(k)]  # k个不同的盐值
   ```

   **编码过程（encode）：**
   - 对每个键值对 `(key, value)`：
     - 用 k 个不同的盐值计算 k 个哈希位置：`pos_i = hash(key + salt_i) % m`
     - 对每个位置做 XOR：`table[pos_i] ^= value`
   - 优点：多个元素可能共享同一位置，但 XOR 保证了可解码性

   **解码过程（decode）：**
   - 对查询的 `key` 计算 k 个位置
   - 对这 k 个位置的值做 XOR：`result = XOR(table[pos_1], table[pos_2], ..., table[pos_k])`
   - 若 key 存在于原始输入，则能恢复原始 value

3. **PSI 交集协议（psi_intersection）**
   ```python
   def psi_intersection(setA: set, setB: set) -> set:
       # 1. 双方使用相同 PRF 密钥
       k = _get_protocol_key()

       # 2. A 方：构建 KV 对并编码为 OKVS
       kv_A = _build_kv_pairs(setA, k, k+1)
       okvs_A = LightweightOKVS()
       okvs_A.encode(kv_A)

       # 3. B 方：构建 KV 对并编码为 OKVS
       kv_B = _build_kv_pairs(setB, k, k+1)
       okvs_B = LightweightOKVS()
       okvs_B.encode(kv_B)

       # 4. 交集判定
       # 原理：交集元素 x 满足 vA(x) == vB(x)，即 vA(x) ^ vB(x) == 0
       for x in setB:
           vA = okvs_A.decode(x)
           vB = okvs_B.decode(x)
           if vA == vB:
               intersection.add(x)
       return intersection
   ```

   **安全性说明：**
   - 攻击者只能看到 OKVS 哈希表（包含 XOR 后的值）
   - 不知道 PRF 密钥，无法反推原始元素
   - 无法判断哈希表中每个位置存储的是哪个元素

4. **带权重版本（psi_intersection_weighted）**
   - 返回交集集合 + 每个元素的权重（0-1）
   - 权重计算：`weights[x] = abs(vA(x)) % 100 / 100.0`

**环境变量：**
- `PSI_PROTOCOL_KEY` — 覆盖默认随机密钥（用于测试）

### 2.3 Big Five 性格建模

#### `backend/modules/personality.py` — 性格向量与相似度计算

**Big Five 人格模型（5 维度）：**

| 维度 | 英文名 | 说明 |
|------|--------|------|
| 外向性 | extroversion | 社交能力、活力、热情 |
| 宜人性 | agreeableness | 信任、善良、合作 |
| 尽责性 | conscientiousness | 自律、组织、责任感 |
| 神经质 | neuroticism | 情绪稳定、焦虑倾向 |
| 开放性 | openness | 创造力、好奇心、求知欲 |

**向量存储：**
- 每个维度取值范围：0.0 ~ 1.0（由问卷 1-5 分映射而来）
- 存储到 `user_personality_bigfive` 表

**输入转换：**

   ```python
       # 输入：{"extro": 4, "agreeableness": 3, ...}
       # 输出：{维度: (v-1)/4} 映射到 0-1
   ```

2. **问卷模式（questionnaire）**
   ```python
   def bigfive_from_questionnaire_1to5(answers_1to5: List[int]) -> Dict[str, float]:
       # 输入：10 个问题的答案（每题 1-5）
       # 每维度 2 题，取平均值后映射到 0-1
       # QUESTION_DIM_INDEX 数组定义题目到维度的映射
   ```

**相似度计算：**

1. **L2 归一化**
   ```python
   def _vec_to_norm_and_comp(vec):
       # 返回 (vec_norm, comp_norm)
       # vec_norm: 向量的 L2 归一化版本
       # comp_norm: (1-vec) 的 L2 归一化版本（互补向量）
   ```

2. **相似度 + 互补度混合**
   ```python
   def trait_similarity_mixed(user_pair, other_pair, mode):
       # 计算：
       # - sim = cosine(user_vec, other_vec)  # 相似度
       # - comp = cosine(user_vec, 1-other_vec)  # 互补度
       # - mix = alpha * sim + (1-alpha) * comp
       #
       # alpha 值按模式不同：
       # - friend: 0.7（侧重相似）
       # - team: 0.5（平衡）
       # - love: 0.6（适度相似）
       return mix, sim, comp
   ```

- 线程安全：`_cache_lock` 保护并发访问
- `upsert_personality_bigfive()` 后自动失效缓存

### 2.4 推荐引擎

#### `backend/modules/recommend.py` — 多维度推荐系统

**四维度评分体系：**

| 维度 | 计算方式 | friend 权重 | team 权重 | love 权重 |
|------|----------|-------------|-----------|-----------|
| 好友 Jaccard | PSI 交集 / 并集 | 40% | 20% | 10% |
| 兴趣相似度 | 语义向量余弦相似度 | 40% | 10% | 50% |
| 技能相似度 | 语义向量余弦相似度 | 0% | 50% | 10% |
| 性格匹配 | Big Five 混合相似度 | 20% | 20% | 30% |

**综合分计算：**
```python
total = w_f * f_score + w_i * i_score + w_s * s_score + w_t * trait_mix
```

**语义匹配（SentenceTransformer）：**

```python
def _semantic_set_similarity(set_a, set_b):
    # 1. 将每个标签向量化（A 和 B 的标签集合）
    A = _encode_tags(a_tags)  # shape: (len(a), embedding_dim)
    B = _encode_tags(b_tags)   # shape: (len(b), embedding_dim)

    # 2. 计算余弦相似度矩阵
    sim = A @ B.T  # shape: (len(a), len(b))

    # 3. 对 A 中每个标签取 B 中的最大相似度
    max_sim = sim.max(axis=1)  # shape: (len(a),)

    # 4. 分数 = 平均最大相似度
    score = max_sim.mean()

    # 5. 共同标签数 = 超过阈值的标签数量
    common_cnt = (max_sim >= threshold).sum()

    return score, common_cnt
```

**推荐历史快照：**
- 每次推荐（skip=0）自动写入 `recommendation_history` 表
- 存储 JSON 格式的推荐结果（包含综合分、维度分、兴趣/技能预览）
- 支持历史查询和删除

**推荐接口：**

1. `GET /api/recommend/<mode>` — 简单推荐（仅返回 ID 和用户名）
2. `GET /api/recommend/<mode>/detailed` — 详细推荐（包含维度分、推荐理由、标签预览）
3. `GET /api/recommend/history` — 推荐历史列表
4. `DELETE /api/recommend/history/<hid>` — 删除单条历史

- 附加信息：成为好友时间、在线态、最近互动徽章、圈子活跃徽章
- 在线态模拟：`_fake_online(fid)` 根据好友 ID 和当前小时生成稳定值

**快捷互加（POST /api/friends/add）：**
- 输入：{friend_id}
- 双向插入 friendships 表
- 无需请求，直接成为好友

**好友请求流程：**

1. **创建请求（POST /api/friend-requests）**
   - 检查：不能向自己发请求、用户是否存在、是否已是好友
   - 已有请求时：pending 状态报错、accepted 报错、rejected 状态可重新发起

2. **收件箱（GET /api/friend-requests/incoming）**
   - 返回：请求 ID、发送者 ID、用户名、状态、创建时间

3. **发件箱（GET /api/friend-requests/outgoing）**

4. **同意请求（POST /api/friend-requests/<rid>/accept）**
   - 检查：请求存在、状态为 pending、接收者是当前用户
   - 更新状态为 accepted
   - 双向插入 friendships 表

5. **拒绝请求（POST /api/friend-requests/<rid>/reject）**
   - 更新状态为 rejected

## 5. 推荐算法详解

### 5.1 推荐流程

```
用户点击推荐页
    ↓
前端调用 GET /api/recommend/{mode}/detailed
    ↓
后端执行推荐算法：
    1. 加载用户数据（好友、兴趣、技能、性格）
    2. 读取隐私偏好，过滤不参与的维度
    3. 遍历所有候选用户：
       - 计算 PSI 好友交集 → friend_score (Jaccard)
       - 计算语义兴趣相似度 → interest_score
       - 计算语义技能相似度 → skill_score
       - 计算 Big Five 性格匹配 → trait_score
       - 加权求和 → total_score
    4. 按 total_score 降序排序
    5. 分页返回结果
    ↓
前端渲染推荐卡片（显示综合分、四维度分、推荐理由）
    ↓
用户点击"加好友" → POST /api/friend-requests
```

### 5.2 权重配置

```python
def _mode_weights(mode):
    if mode == "friend":
        return 0.4, 0.4, 0.0, 0.2  # (好友, 兴趣, 技能, 性格)
    if mode == "team":
        return 0.2, 0.1, 0.5, 0.2
    if mode == "love":
        return 0.1, 0.5, 0.1, 0.3
    return 0.4, 0.4, 0.0, 0.2
```

### 5.3 隐私保护机制

1. **PSI 交集计算**
   - 好友/兴趣/技能交集使用 OKVS-PSI 原型
   - 不传输明文集合

2. **隐私开关**
   - 用户可关闭任意维度的数据分享
   - 关闭后该维度在推荐中不参与（分数为 0）

3. **关系分级展示**
   - 陌生人只能看到基础信息和标签数量
   - 好友可查看完整兴趣技能

## 6. 维护建议

### 6.1 性能优化

| 问题 | 建议方案 |
|------|----------|
| 推荐计算慢 | 增加缓存层，预计算用户向量 |
| 列表渲染卡顿 | 虚拟列表（如 FriendsView） |
| 语义模型加载慢 | 模型预加载，进程启动时初始化 |
| 聊天实时性差 | WebSocket 替代轮询 |

### 6.2 安全建议

| 问题 | 建议 |
|------|------|
| 短信验证码 | 替换为真实短信服务，增加限流 |
| Session 安全 | 生产环境设置强随机 FLASK_SECRET_KEY |
| CORS | 收紧 CORS_ORIGINS，仅允许自有域名 |

### 6.3 功能扩展

| 方向 | 说明 |
|------|------|
| 真实 PSI | 升级为专业 PSI 库（如 pyspiral、libOTe） |
| 推荐多样性 | 引入 MMR（最大边际相关性）避免推荐重复 |
| 推荐可解释性 | 详细展示每个维度的贡献度 |
| WebSocket 聊天 | 替代轮询，提升实时性 |
